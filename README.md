# Рукогон

`Рукогон` — сокращение от `Руководитель Гонки`.

Приложение полностью сгенерировано ИИ.

Статическая аркадная игра с видом сверху на `React + Vite`.

Игрок управляет руководителем дрифт-соревнования и должен удерживать площадку под контролем: тушить инциденты на трассе, разбирать документы в штабе, реагировать на проблемы на трибунах, у кассы и в паддоке. Игра заканчивается, если проблем становится слишком много или хотя бы одна из них "сгорает".

Приложение полностью работает на клиенте. Серверная часть не требуется: серверу достаточно раздавать статические файлы. Настройки, статистика и пользовательская карта сохраняются в `localStorage`.

## Особенности

- полностью статическое веб-приложение без внешних API
- `canvas`-рендеринг и пиксель-арт стилистика
- 5 игровых зон: трасса, штаб, трибуны, касса, паддок
- разные типы проблем со своими анимациями и визуальными иконками
- прогресс ожидания проблемы и "сгорание" при критическом затягивании
- подвижные проблемы-боты, которых нужно догонять
- фоновые NPC: фотограф, человек с рацией, блогер
- короткие реплики над активными проблемами
- редактор карты с тайлами, перетаскиванием подписей зон и проверкой валидности
- импорт и экспорт карт в JSON
- несколько режимов управления: клавиатура, мышь, автобот
- сохранение рекорда, количества сессий и пользовательских опций

## Публикация

Для публикации нужен только каталог `dist/`.

Типовой сценарий:

1. Выполнить `npm install`
2. Выполнить `npm run build`
3. Скопировать содержимое `dist/` на сервер

Проект подходит для любого статического хостинга или обычного веб-сервера, который умеет раздавать HTML, CSS, JS и изображения.

Примечание: в `vite.config.js` задан `base` для развёртывания в подкаталог `/rukogon/`. Если приложение будет размещаться по другому пути, значение `base` нужно изменить перед сборкой.

## Игровая логика

На карте есть 5 зон:

- Трасса
- Штаб
- Трибуны
- Касса
- Паддок

Типы проблем:

- происшествия на трассе
- документы в штабе
- события на трибунах
- проблемы у кассы
- споры в паддоке

Особенности правил:

- игрок ходит только по проходимым тайлам карты
- паддок является проходимой зоной
- часть проблем может быть подвижной
- у каждой проблемы есть таймер ожидания
- с каждой решённой проблемой игра постепенно усложняется
- в игре есть фоновая жизнь: NPC перемещаются по карте и создают дополнительный визуальный шум

## Управление

### Клавиатура

- `W`, `A`, `S`, `D` или стрелки — движение
- `Space` — старт или пауза
- `Enter` — начать новую игру
- `Esc` — завершить текущую игру

### Кнопки интерфейса

- `Старт / Стоп` — начать новый заезд или завершить текущий
- `Пауза / Продолжить` — управлять паузой
- `Режим редактирования` — включить редактор карты

### Переключаемые опции

- `Звук`
- `Автобот`
- `Мышь`
- `Подвижные проблемы`
- `NPC`

### Управление мышью

Если опция `Мышь` включена, персонаж бежит в сторону курсора на игровом поле. В режиме редактора мышь используется для рисования карты и перетаскивания подписей зон.

## Редактор карты

Редактор позволяет:

- выбирать категорию тайла
- рисовать по карте кликом и ведением мыши
- перемещать подписи зон
- сохранять карту в JSON
- загружать карту из JSON
- возвращаться к карте по умолчанию

Карта по умолчанию загружается из файла [public/tracks/default.json](/Users/nich/develop/rds/рукогон/public/tracks/default.json).

### Формат файла карты

Файл карты содержит:

- `version`
- `mapRows`
- `labelPositions`

Пример структуры:

```json
{
  "version": 1,
  "mapRows": ["GGGG", "GAPP", "GDDD"],
  "labelPositions": {
    "headquarters": { "x": 120, "y": 130 },
    "stands": { "x": 810, "y": 92 },
    "ticketOffice": { "x": 118, "y": 418 },
    "paddock": { "x": 760, "y": 455 },
    "track": { "x": 448, "y": 58 }
  }
}
```

## Структура проекта

```text
src/
  components/    React-компоненты интерфейса и canvas-контейнер
  game/          игровая логика, карта, правила, рендер, состояние
  hooks/         пользовательские хуки
  styles/        глобальные и прикладные стили
public/
  assets/        локальные изображения и портреты
  tracks/        JSON-файлы карт
```

## Локальное хранение

В `localStorage` сохраняются:

- лучший результат
- количество игровых сессий
- состояние опций
- пользовательская карта и позиции подписей
